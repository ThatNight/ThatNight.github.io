Android 系统启动流程

本文基于Android 7.0分析

流程

  1. BootLoader加载(系统开始前的一个小程序，将OS拉起来并运行)
  2. Kernel加载（文件系统挂载，init进程启动，设备初始化服务）
  3. init进程启动（解析init.rc）
  4. Zygote进程启动
  5. SystemServer启动服务(AMS,PMS,WMS,PowerMS,AccoutMS,IputMethodMS…)

  • 不过我们一般都是直接从下面的过程开始：

    init->Zygote->SystemServer->应用程序

init进程

  • 总结

    1. 创建一些文件夹挂载设备
    2. 初始化和启动属性服务
    3. 解析init.rc配置文件并启动Zygote进程

  • init.rc：内部由Android初始化语言编写的脚本，一个配置文件；

  • 解析Service：解析service的rc文件，创建出Service对象，根据选项域的内容填充Service对象，最后将Service加入到vector类型的Service链表中；
  • init启动Zygote:解析init.zygote.rc文件创建的，对应的源文件是App_main.cpp，主要是调用AppRuntime的start方法来启动Zygote的
  • 属性服务：类似Windows的注册表，采用键值对来记录软件的一些信息，系统或软件重启时会根据这些信息进行初始化工作；==这里面会创建一个非阻塞的Socket给最多8个试图设置属性的用户提供服务；

Zygote

  • 简介

    Android系统中DVM、应用程序进程以及运行系统的关键服务的SystemServer进程都是由Zygote进程来创建的，所以也被称为‘孵化器’；通过fork（复制进程）的形式来创建应用程序进程和SystemServer进程的；

    由于Zygote启动时会启动DVM（Dalvik），因此通过fork而创建的进程可以在内部获取一个DVM的实例拷贝；

  • AppRuntime

    在init进程的最后会调用AppRuntime的start方法启动Zygote的；

    • AppRuntime 主要工作
      • 创建Android运行时环境
      • 启动SystemServer
        App Runtime继承于AndroidRuntime，所以着重于AndroidRuntime；
    • AndroidRuntime主要工作
      • 启动Java Dalvik虚拟机，并设置虚拟机相关参数
      • 注册JNI函数，后续Java世界用到的一些函数都是采用native方式来实现的
      • 调用ZygoteInit.main方法正式进入Java世界

  • ZygoteInit.main

    • 总结

      • 创建IPC通信的Socket
      • 预加载类和资源
      • 启动SystemServer
      • 监听其他进程发出来的请求

    • registerZygoteSocket

      创建Local Server Socket，也就是服务端的Socket；当SystemServer启动后，这个Socket等待ActivityManagerService请求Zygote进程来创建新的应用程序进程；

    • 启动SystemService

      • 创建一个String类的args数组：保存启动SystemServer的参数，进程的用户id和用户组id被设置为1000；并且拥有用户组1001~1010、1018、1021、1032、3001~3010的权限；
      • 将args数组封装成Arguments对象并被forkSystemServer函数调用；
      • forkSystemServer：通过fork创建一个子进程，如果返回的pid（process id）为0，表示在新创建的子进程执行；
      • 调用handleSystemServerProcess启动SystemServer进程；

    • runSelectLoop

      启动SystemServer后，最后进入runSelectLoop函数

      • 调用第一步注册的socket.getFileDescriptor()来获得Socket的fd字段的值并添加在fd列表fds中；
      • 无限循环等待ActivityManagerService请求Zygote进程来创建新的应用程序

      • 遍历将fds存储的信息转到pollFds数组中，最后在进行对pollFds遍历，如果i==0则说明服务端Socket与客户端连接上了，也就是当前Zygote进程与ActivityManagerService建立了连接；

      • 然后调用acceptCommandPeer得到ZygoteConnection类添加到Socket连接表peers中，接着将该Zygote Connection的fd添加到fd列表fds中；以便可以接收到ActivityManagerService发送过来的请求；

      • 如果i>0则说明AMS向Zygote进程发送了一个创建应用进程的请求，然后调用ZygoteConnection的runOnce函数来创建一个新的进程，并成功创建后将这个连接从Socket连接列表peers和fd列表fds中清除；

  • 总结

    • 创建AppRuntime并调用start启动Zygote进程
    • 创建DVM并为DVM注册JNI环境
    • 通过JNI调用ZygoteInit的main函数进入Zygote的java框架层
    • 通过registerZygoteSocket创建服务端socket，通过runSelectLoop函数等待AMS的请求来创建新的进程
    • 启动System Server进程

SystemServer

  • 简介

    Android系统几乎所有核心的Service都在这个进程中，启动流程如下：

    紫色的运行在Zygote的进程，蓝色的是运行在新创建的system_server，这是fork机制实现的（fork会返回2次）；

  • 流程

    1. Zygote启动SystemServer进程
      上面Zygote也讲了，在startSystemServer中调用了handleSystemServerProcess来启动；

      如果创建system_server失败时，需要重启zygote进程；对于5.0以上的系统，有两个Zygote进程（Zygote、Zygote64），一般来说64位系统的父进程时Zygote64进程；

      1. 当kill system_server，至重启Zygote64和system_server;
      2. 当kill zygote64，同上
      3. 当kill zygote，则重启全部进程

    2. handleSystemServerProcess方法流程

      1. SystemServer进程是fork了Zygote进程的地址空间，因此也会得到Zygote的Socket，这个socket对SystemServer没有用处，所以要关闭该socket；
      2. 调用RuntimeInit的zygoteInit函数，里面调用了nativeZygoteInit，主要初始化Zygote

    3. 启动Binder线程池

      最后一行代码时启动一个Binder线程池的， 这样SystemServer进程就可以使用Binder与其他进程来通信了；RuntimeInit.java的nativeZygoteInit函数主要做的时启动Binder线程池

    4. invokeStaticMain
      上面的RuntimeInit.java注释2调用了applicationInit函数；而方法实现时调用了invokeStaticMain函数；

      invokeStaticMain里面主要通过反射返回SystemServer类，然后找到main函数，将main函数传入到MethodAndArgsCaller异常中并抛出该异常，截获MethodAndArgsCaller异常的代码在ZygoteInit的main中；

  • 解析SystemServer进程

    • 主要代码

    • 启动引导服务

      1. 阻塞等待与installer创建
      2. 启动ActivityManagerService
      3. 启动PowerManagerService并初始化
      4. 启动LightsService
      5. 启动DisplayManagerService
      6. 启动PackageManagerService
      7. 启动UserManagerService
      8. 启动传感器服务
    • 启动核心服务
      1. 启动BatteryService，统计电量
      2. 启动UsageStatsService，统计应用使用情况
      3. 启动WebViewUpdateService
    • 启动其他服务
      1. 启动WindowManagerService
      2. …

  • 各启动阶段

    再Phase1000 ，调用AMS.finishBooting()，系统服务启动阶段完成，system_server进程启动完成进入Looper.loop()状态待命，等待消息队列MessageQueue中的消息到来；

  • 总结

    • 启动Binder线程池，这样可以与其他进程通信
    • 创建SystemServiceManager用于对系统的服务进行创建、启动和生命周期管理
    • 启动各种系统服务
